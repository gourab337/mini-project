About the Binding Kit

The IndigoVision Binding Kit enables binding software to be developed that integrates external systems such as alarm management and access control systems with IndigoVision Control Center. It enables developers to write applications that forward external alarm events to, or receive zone (or detector) notifications from, an IndigoVision Alarm Server (NVR-AS). Zone notifications can notify state changes such as alarmed, acknowledged or cleared. IvBind provides a C API as well as a COM interface to allow scripts or applications to be written in a wide variety of programming languages. Refer to the help files in the docs directory for further information.

Installation Instructions

Before the IvBind2 COM interface can be used, IvBind2.dll must be extracted from IvBind2.zip on the IndigoVision Product CD then registered with Windows. To register the IvBind2 COM interface with Windows, do the following:

On 32-bit operating systems

  1. Open a Command Prompt as an administrator and navigate to the directory that you extracted IvBind2.dll to.
  2. Type regsvr32.exe ivbind2.dll then Enter.

On 64-bit operating systems

  1. Open a Command Prompt as an administrator and navigate to the directory that you extracted IvBind2.dll to.
  2. Type C:\Windows\SysWOW64\regsvr32.exe ivbind2.dll then Enter.
  3. Navigate to the ivbind2\registry directory.
  4. To enable use of DllHost.exe as a surrogate run comsurrogate.reg and accept the warnings from windows regarding the changes to your registry.

Important Information about Binding Kit 2.14

NVR-AS Compatibility

Binding Kit 2.14 has been designed to operate with NVR-AS 15.2 and later and can query all detector types from those versions. It is not recommended to use Binding Kit 2.13 with older NVR-AS versions as not all detector types would be seen. Specifically ONVIF Basic Analytics and ONVIF Digital Input detector types would not be seen.

What's new in Binding Kit 2.14

DNA Proxy Support

The Binding Kit SDK can now communicate with Alarm Servers via the IndigoVision DNA Proxy in deployments where there is no direct network connectivity. To use a DNA Proxy, call SetDnaProxy before communicating with the Alarm Server. If a DNA Proxy is not required, no updates are necessary.

The IvZoneMonitor and IvDetectorMonitor samples have been updated to demonstrate the use of SetDnaProxy.

Known Issues

  • NVR Fault detectors will only report a change to the detector when a new fault condition occurs or when all faults are cleared. If a fault condition clears but other fault conditions remain no update notification will be received.
